# 摘要

本课程设计实现了一个“校园最短路径导航系统”。系统以校园主要地点为顶点、地点之间的路径长度为边，使用**图结构**进行存储，并采用 **Dijkstra 算法**计算任意两点间的最短路径及距离。系统由 C 语言实现的命令行核心模块与纯静态 HTML/CSS/JavaScript 前端原型两部分组成：C 端负责数据初始化、最短路径计算与 `routes.json` 导出；前端负责可视化展示路径文字、校园地图与数据管理界面。全文从需求分析、概要设计、详细设计、编码实现、测试分析等方面对系统进行了完整说明。

**关键词**：最短路径；Dijkstra 算法；图结构；邻接表；课程设计

## 1.2 设计内容与要求

根据任务书要求，设计实现一个校园最短路径导航系统，具体内容包括：

1. 输入校园主要地点（如教学楼、食堂、宿舍、图书馆等）及路径长度；
2. 使用**图结构**存储地点与路径；
3. 支持查询任意两点间的最短路径及距离；
4. 必须实现 **Dijkstra** 或 **Floyd** 算法；

6. 处理异常输入，如不存在的节点、起终点相同等。

设计要求：符合课题要求并实现相应功能；界面友好美观、操作方便易行；注意程序的实用性与安全性。

# 第二章 需求分析

## 2.1 功能需求

### 2.1.1 数据录入与维护

系统需要维护校园地点与路径数据。每个地点包含编号、名称、类型、图标及二维坐标；每条路径记录起点、终点与路径长度（米）。本系统采用内置示例数据，覆盖教学楼、图书馆、食堂、宿舍区、体育馆、实验楼、校医院、行政楼、操场等 9 个主要地点，以及 15 条双向路径。

### 2.1.2 最短路径查询

用户选择起点与终点后，系统应返回两点之间的最短路径及总距离。查询结果以文字形式输出，例如：

```
教学楼 → 行政楼 → 宿舍区
最短距离：350 米
```

### 2.1.3 结果可视化

前端原型通过校园地图与路径卡片两种方式展示结果：地图中高亮显示经过的节点与边，卡片中列出路径文字、分段距离及总距离。

### 2.1.4 异常处理

系统需对以下异常输入进行处理并给出明确提示：

- 起点或终点编号不存在；
- 输入非数字；
- 起点与终点相同；
- 功能菜单选择非法。

## 2.2 非功能需求

1. **正确性**：最短路径计算结果与 Dijkstra 算法理论结果一致；
2. **易用性**：命令行菜单简洁，前端界面直观；
3. **可扩展性**：数据与算法分离，便于后续替换为文件输入或后端接口；
4. **可维护性**：代码模块化，遵循 Conventional Commits 进行版本管理。

# 第四章 详细设计

## 4.1 数据结构设计

### 4.1.1 地点枚举

地点使用枚举类型 `PlaceId` 与数组下标一一对应，便于代码阅读与维护：

```c
typedef enum {
    PLACE_TEACHING = 0,
    PLACE_LIBRARY,
    PLACE_CANTEEN,
    PLACE_DORM,
    PLACE_GYM,
    PLACE_LAB,
    PLACE_HOSPITAL,
    PLACE_OFFICE,
    PLACE_PLAYGROUND,
    PLACE_COUNT
} PlaceId;
```

### 4.1.2 边结构体

边采用有向记录存储，无向图通过成对添加两条有向边实现：

```c
typedef struct {
    int from;       /* 起点 PlaceId */
    int to;         /* 终点 PlaceId */
    int distance;   /* 路径长度，单位米 */
} Edge;
```

### 4.1.3 图结构体

图采用**邻接表 + 索引表**的混合结构：

```c
typedef struct {
    const char *ids[PLACE_COUNT];
    const char *names[PLACE_COUNT];
    const char *types[PLACE_COUNT];
    const char *icons[PLACE_COUNT];
    int  coord_x[PLACE_COUNT];
    int  coord_y[PLACE_COUNT];

    Edge edges[MAX_EDGES];
    int  edge_count;
} Graph;
```

其中 `edges` 数组存储所有有向边，通过 `graph_degree` 与 `graph_edge_at` 按起点索引获取邻接边，形成邻接表效果。该结构空间紧凑，便于遍历与扩展。

### 4.1.4 最短路径结果结构体

```c
typedef struct {
    int distance;
    int path[DIJKSTRA_MAX_PATH];
    int path_len;
} DijkstraResult;
```

## 4.2 算法设计

### 4.2.1 Dijkstra 算法

系统采用**朴素 Dijkstra 算法**，适用于非负权无向图。由于本系统地点数 `PLACE_COUNT = 9`，规模较小，O(V²) 的时间复杂度完全满足需求。

算法步骤：

1. 初始化距离数组 `dist[]` 为无穷大，`visited[]` 为假，`prev[]` 为 -1；起点 `dist[start] = 0`。
2. 每次在未访问顶点中选择 `dist` 最小的顶点 `u`。
3. 若 `u` 为终点或不存在可达顶点，则结束。
4. 标记 `u` 已访问，松弛其所有邻接边：若 `dist[u] + w(u, v) < dist[v]`，则更新 `dist[v]` 并记录前驱 `prev[v] = u`。
5. 重复步骤 2~4，直到所有顶点处理完毕。
6. 从终点回溯 `prev[]` 得到最短路径，反转为正序输出。

### 4.2.2 复杂度分析

- **时间复杂度**：O(V²)，其中 V 为顶点数。每次选择最小距离顶点 O(V)，共 V 轮；每轮松弛邻接边 O(V)。
- **空间复杂度**：O(V + E)，用于存储距离、访问标记、前驱数组与边表。

### 4.2.3 Floyd 算法说明

虽然本系统未采用 Floyd 算法，但该算法同样适用于本场景。Floyd 通过动态规划计算所有顶点对之间的最短路径，时间复杂度为 O(V³)，更适合稠密图或需要频繁查询全部点对的场景。本系统选择 Dijkstra 是因为只需按需计算单源最短路径，实现更简洁。

## 5.3 Dijkstra 算法实现

`src/dijkstra.c` 实现了朴素 Dijkstra 算法，并通过前驱数组 `prev[]` 重构路径：

```c
DijkstraResult dijkstra_shortest_path(const Graph *g, int start, int end)
{
    /* 初始化 */
    for (int i = 0; i < PLACE_COUNT; i++) {
        dist[i]    = INT_MAX;
        visited[i] = false;
        prev[i]    = -1;
    }
    dist[start] = 0;

    for (int iter = 0; iter < PLACE_COUNT; iter++) {
        int u = -1, best = INT_MAX;
        for (int i = 0; i < PLACE_COUNT; i++)
            if (!visited[i] && dist[i] < best) { best = dist[i]; u = i; }

        if (u == -1 || dist[u] == INT_MAX) break;
        if (u == end) break;
        visited[u] = true;

        int deg = graph_degree(g, u);
        for (int k = 0; k < deg; k++) {
            Edge e = graph_edge_at(g, u, k);
            int v = e.to;
            if (visited[v]) continue;
            long candidate = (long)dist[u] + (long)e.distance;
            if (candidate < dist[v]) {
                dist[v] = (int)candidate;
                prev[v] = u;
            }
        }
    }

    /* 回溯并反转得到正序路径 */
    /* ... */
    return result;
}
```

算法通过 `long` 类型计算候选距离，避免 `INT_MAX` 相加导致的整数溢出。

## 5.4 JSON 导出与前端联动

`src/main.c` 支持 `--export` 参数，预计算所有 `i ≠ j` 的最短路径并写入 `assets/data/routes.json`，同时同步到前端开发用的 mirror 目录：

```c
if (export_mode) {
    if (export_routes_json(&g, EXPORT_PATH) != 0) return 1;
    if (strcmp(EXPORT_PATH, EXPORT_MIRROR_PATH) != 0) {
        return export_routes_json(&g, EXPORT_MIRROR_PATH);
    }
    return 0;
}
```

导出的 JSON 包含 `places` 与 `routes` 两个数组，前端通过 `fetch` 加载后建立 `routesIndex` 映射，实现 O(1) 查询。

## 5.5 前端交互实现

前端使用纯静态技术栈。核心逻辑位于 `js/app.js`：

- `loadCProgramData()`：异步加载 `data/routes.json`；
- `dijkstra(start, end)`：浏览器端兜底算法，与 C 端结果一致；
- `renderMap(activePath)`：基于 SVG 渲染校园地图与高亮路径；
- `initAddPlaceDialog()`：演示性新增地点弹窗。

主题切换通过 `data-theme` 属性与 CSS 变量实现，并加入 250ms 平滑过渡。
