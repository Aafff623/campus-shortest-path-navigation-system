# Domain docs

This repo uses a **single-context** layout.

## Context file

- `CONTEXT.md` at the repo root describes the project domain, terminology, and boundaries.

## Architecture Decision Records

- `docs/adr/` contains architecture decision records.
- Name them `YYYYMMDD-title.md`.

## Consumer rules

- Read `CONTEXT.md` before suggesting architecture or design changes.
- Check `docs/adr/` before contradicting a previous decision.
- Keep domain language consistent across issues and code.
