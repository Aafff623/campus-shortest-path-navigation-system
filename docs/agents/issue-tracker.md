# Issue tracker

This repo uses **local markdown** for issue tracking.

## Where issues live

- `.scratch/<feature-or-issue-name>/`
- Each issue is a markdown file, e.g. `.scratch/shortest-path-algo/README.md` or `.scratch/shortest-path-algo/spec.md`

## Creating an issue

1. Make a directory under `.scratch/` named after the feature or problem.
2. Write a markdown file inside describing the bug, feature, or task.
3. Add triage label metadata in the front matter or body.

## Conventions

- Keep issues focused and actionable.
- Link related issues by relative path.
- Close an issue by deleting or archiving the directory when done.
