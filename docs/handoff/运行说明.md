# 运行说明

## 环境要求

- Windows 10/11 或兼容 POSIX 的环境
- gcc 或 MinGW-w64（`gcc --version` 能正常输出）
- make（Windows 下可用 MinGW 自带的 `mingw32-make`，或安装 GNU Make）

## 目录结构准备

```bash
assets/data/   # 用于存放 C 程序导出的 routes.json
bin/           # 编译产物
src/           # C 源码
```

## 编译

```bash
make build
```

或手动：

```bash
gcc src/*.c -o bin/campus_nav.exe
```

## 运行交互模式

```bash
make run
```

或手动：

```bash
./bin/campus_nav.exe
```

## 导出 JSON 供前端使用

```bash
make export
```

或手动：

```bash
./bin/campus_nav.exe --export
```

导出成功后，`assets/data/routes.json` 会被更新。

## 运行前端页面

前端为纯静态页面，可直接用浏览器打开 `assets/prototype/campus-nav-prototype/index.html`。

推荐使用 VS Code 的 Live Server 插件，或 Python 简单 HTTP 服务器（以下使用 8081 端口，避免与本地其他服务冲突）：

```bash
cd assets/prototype/campus-nav-prototype
python -m http.server 8081
```

然后在浏览器访问 `http://localhost:8081`。

如 8081 也被占用，可换成其他端口，例如 `python -m http.server 8082`，并相应调整访问地址。

### 2D 校园图 overlay

前端会自动加载 `assets/prototype/campus-nav-prototype/data/campus-layout.json`，并使用其中的 `map.image` 作为底图（当前为官方校园平面图 `images/nuc-campus-map-official.jpg`）。

如需校准坐标：

1. 启动本地服务器（见上文，例如端口 8081）。
2. 通过主导航进入 **坐标标定** 页面，或浏览器打开 `http://localhost:8081/calibrate.html`。
3. 在官方图上点击各个地点的建筑中心，记录坐标。
4. 更新 `data/campus-layout.json` 中的 `places` 与 `edgePolylines`。
5. 刷新前端页面查看效果。

> 坐标标定页通过 iframe 嵌入 `tools/coordinate-debug-tool.html`。如需单独打开工具，也可直接访问 `http://localhost:8081/tools/coordinate-debug-tool.html`。
