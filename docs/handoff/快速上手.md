# 快速上手 — 在另一台电脑从零运行

面向第一次接手、换机部署或答辩演示的读者：从一台**什么都没装**的电脑开始，一步步把项目跑起来。

> 命令逐条说明见同目录 [`运行说明.md`](运行说明.md)；本文只讲「从零到能跑」的完整路径与常见坑，不重复细节。

---

## 0. 需要装什么

| 工具 | 用途 | 验证命令 |
|---|---|---|
| MinGW-w64（含 `gcc` + `mingw32-make`） | 编译 C 后端 | `gcc --version` |
| Python 3 | 起前端本地服务器 | `python --version` |
| Git（可选） | 拉取仓库；不装也可下载 zip | `git --version` |

> 只想看前端界面、不改后端？也得至少跑一次导出生成 `routes.json`；或直接用仓库里已有的 `assets/data/routes.json`，跳到 [第 3 步](#3-启动前端)。

---

## 1. 安装环境（Windows）

### 1.1 MinGW-w64（提供 gcc）

最简单的方式是 **winlibs 解压即用**：

1. 打开 <https://winlibs.com/>，下载 **UCRT runtime / Win64** 的 zip（含 GCC）。
2. 解压，得到 `mingw64` 文件夹，放到例如 `C:\mingw64`。
3. 把 `C:\mingw64\bin` 加入系统 `Path`：
   *设置 → 系统 → 系统信息 → 高级系统设置 → 环境变量 → 选中 `Path` → 编辑 → 新建 → 粘贴 `C:\mingw64\bin` → 一路确定*。
4. **新开**一个 PowerShell（让 PATH 生效），运行：
   ```powershell
   gcc --version
   mingw32-make --version
   ```
   两条都有版本号输出即成功。

> 备选：装 MSYS2 后执行 `pacman -S mingw-w64-ucrt-x86_64-gcc make`，再把对应 `bin` 加入 PATH。

### 1.2 Python 3

1. 打开 <https://www.python.org/downloads/> 下载并安装。
2. 安装第一屏务必勾选 **Add python.exe to PATH**。
3. 新开终端验证：
   ```powershell
   python --version
   ```

---

## 2. 获取项目

**有 Git：**

```bash
git clone https://github.com/Aafff623/campus-shortest-path-navigation-system.git
cd campus-shortest-path-navigation-system
```

**没有 Git：** 打开仓库页面 → 绿色 **Code** 按钮 → **Download ZIP** → 解压 → 进入解压后的目录。

---

## 3. 编译并导出数据

在项目根目录执行：

```bash
make export
```

> ⚠️ MinGW 自带的 make 通常叫 `mingw32-make`。若提示 `make 不是内部或外部命令`，改用：
> ```bash
> mingw32-make export
> ```
> 完全不想用 make，可手动两步：
> ```bash
> gcc src/*.c -o bin/campus_nav.exe
> bin/campus_nav.exe --export
> ```

成功后会更新 `assets/data/routes.json`（前端的数据来源）。

---

## 4. 启动前端

```bash
cd assets/prototype/campus-nav-prototype
python -m http.server 8081
```

浏览器打开 <http://localhost:8081/index.html>。

> ⚠️ **不要直接双击 `index.html`。** 前端通过 `fetch` 读取 `routes.json`，在 `file://` 下会因跨域被浏览器拦截，只能退回内置 fallback 数据。必须用上面的本地服务器走 `http://localhost`。

---

## 5. 验证跑通

1. 进入 **路径查询** 页。
2. 选一个起点、一个终点，点 **查询**。
3. 看到**文字路线 + 总距离 + 地图高亮**，即全链路（C 导出 → JSON → 前端渲染）成功。

---

## 常见坑速查

| 现象 | 原因 | 解决 |
|---|---|---|
| `make 不是内部或外部命令` | MinGW 的 make 叫 `mingw32-make` | 用 `mingw32-make export`，或手动 gcc 两步 |
| `gcc 不是内部或外部命令` | PATH 没配好，或终端没重开 | 把 `C:\mingw64\bin` 加进 Path，**新开**终端再试 |
| 双击 `index.html` 查询结果不对/为空 | `file://` 下 `fetch routes.json` 跨域失败，走了 fallback | 必须用 `python -m http.server` 起，从 `http://localhost` 打开 |
| `端口 8081 被占用` | 其他服务占用该端口 | 换端口：`python -m http.server 8082`，访问地址同步改 |
| 改了 C 代码但前端没变化 | 没重新导出 JSON | 重新 `make export`（或 `mingw32-make export`）再刷新页面 |
| 地图坐标想微调 | 官方图坐标需人工标定 | 见 [`运行说明.md`](运行说明.md) 的「2D 校园图 overlay / 坐标标定」 |
