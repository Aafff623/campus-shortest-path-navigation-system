# Plan — 课程设计说明书

> 任务 #9
> 日期：2026-06-23

## 1. 目标

按任务书要求撰写《校园最短路径导航系统》课程设计说明书，最终版放入 `docs/reports/`。

## 2. 交付物

| 文件 | 说明 |
|------|------|
| `docs/reports/课程设计说明书.md` | Markdown 源文件，便于后续修订与版本控制 |
| `docs/reports/课程设计说明书.docx` | Word 交付版，由 Pandoc 从 Markdown 转换 |

## 3. 说明书结构

1. 摘要
2. 第一章 绪论（设计目的、内容与要求）
3. 第二章 需求分析（功能需求、非功能需求、异常处理）
4. 第三章 概要设计（系统架构、功能模块、运行环境）
5. 第四章 详细设计（数据结构、Dijkstra 算法、界面设计）
6. 第五章 编码实现（目录结构、核心代码片段、JSON 导出、前端交互）
7. 第六章 测试（测试环境、功能测试用例、结果分析）
8. 第七章 总结与展望
9. 参考文献
10. 附录 A：Makefile
11. 附录 B：routes.json 结构示例

## 4. 关键内容

- **数据结构**：邻接表（`Graph`、`Edge`、`PlaceId`），与前端原型共享地点/边数据。
- **核心算法**：朴素 Dijkstra，O(V²)，通过 `prev[]` 回溯路径。
- **数据交换**：C 端 `--export` 预计算 9×8=72 条最短路径，写入 `assets/data/routes.json`。
- **前端**：纯静态 HTML/CSS/JS，读取 JSON 后 O(1) 查询，SVG 地图可视化。
- **测试用例**：覆盖正常查询、多跳最短路径、非法编号、起终点相同、非法菜单输入、JSON 导出。
