# PRD — 校园最短路径导航系统

## 1. Summary

本项目是中北大学软件学院《数据结构》课程设计，要求用 **C 语言** 实现一个校园最短路径导航系统。系统以图结构存储校园地点与路径，使用 **Dijkstra 算法** 计算任意两点间的最短路径，并通过命令行/终端界面输出路径文字描述（如 `教学楼 → 图书馆`）。同时提供一个前端静态原型用于展示和截图，最终交付可运行程序、源码及课程设计说明书。

## 3. Background

《数据结构》课程设计要求学生综合运用所学数据结构与算法，完成一个小型应用软件的开发全过程。校园最短路径导航系统是典型的图论应用：将校园地点抽象为图的顶点，道路抽象为带权边，通过最短路径算法为师生提供导航参考。

**为什么现在做？** 这是课程规定的教学环节，起讫日期为 2026 年 6 月 22 日 ~ 2026 年 7 月 3 日。

## 4. Objective

### 总体目标

实现一个可运行的 C 语言程序，能够录入/加载校园地点与路径数据，查询任意两点间最短路径，并给出文字形式的路径输出。

### 关键结果（SMART）

- **KR1**：在 C 程序中定义图的结构体，支持至少 9 个地点和 15 条双向路径的存储。
- **KR2**：实现 Dijkstra 算法，能正确计算任意两点间最短距离与路径序列。
- **KR3**：命令行界面支持地点选择、路径查询、异常输入提示（未选择、相同地点、不存在的节点、不可达）。
- **KR4**：输出文字路径，例如 `教学楼 → 图书馆`，并显示总距离。
- **KR5**：完成课程设计说明书，包含需求分析、概要设计、详细设计、测试用例与总结。

## 5. Market Segment(s)

**目标用户**

- 课程设计验收老师：关注功能完整性、算法正确性、代码规范与文档质量。
- 最终使用者：校园内需要快速了解两点间最短步行路线的师生（演示/截图用途）。

**约束条件**

- 必须使用 C 语言实现。
- 必须使用图结构存储数据。
- 必须实现 Dijkstra 或 Floyd 算法。
- 起讫日期固定，交付物包括可运行程序 + 源码 + 说明书。

## 6. Value Proposition(s)

### 用户获得的价值

- **路径查询**：输入起点和终点即可得到最短路线与距离。
- **异常保护**：对非法输入给出清晰提示，避免程序崩溃。
- **学习价值**：完整覆盖数据结构课程设计中“问题分析 → 数据结构设计 → 算法实现 → 测试”的开发流程。

### 避免的痛苦

- 手工查阅校园地图估算路线。
- 程序因输入错误而异常退出。
- 算法实现无测试、无法验证正确性。

## 7. Solution

### 7.1 UX / 原型

前端静态原型已提供在 `assets/prototype/campus-nav-prototype/`，包含以下页面：

- `index.html`：首页 / 系统入口，展示常用地点与系统概览。
- `query.html`：路径查询页，选择起点和终点。
- `result.html`：查询结果页，展示最短距离、文字路径、地图高亮。
- `locations.html`：地点管理页（演示用途）。
- `paths.html`：路径管理页（演示用途）。
- `algorithm.html`：Dijkstra 算法说明页。
- `about.html`：系统功能说明页。

C 语言程序采用**命令行菜单界面**，对应原型中的核心流程：

1. 展示地点列表。
2. 用户输入起点编号和终点编号。
3. 程序调用 Dijkstra 算法计算。
4. 输出最短距离和路径文字描述。
5. 循环支持多次查询，直到用户退出。

### 7.2 Key Features

| 功能模块 | 说明 | 验收标准 |
|----------|------|----------|
| 地点管理 | 程序内置示例地点（教学楼、图书馆、食堂、宿舍区等 ≥9 个），支持编号选择 | 能正确列出所有地点 |
| 路径管理 | 程序内置示例路径边（≥15 条），每条边有长度且双向通行 | 图结构能正确加载 |
| 最短路径查询 | 使用 Dijkstra 算法计算任意两点间最短路径 | 结果与手动验证一致 |
| 文字路径输出 | 输出 `地点A → 地点B → 地点C` 及总距离 | 格式清晰可读 |
| 异常输入处理 | 处理编号越界、相同起终点、不可达等情况 | 程序不崩溃并给出提示 |
| 地图可视化（前端） | 前端 SVG 地图高亮最短路径 | 用于截图和说明书配图 |

### 7.3 Technology

- **实现语言**：C（C99 标准）
- **数据结构**：邻接矩阵或邻接表存储无向带权图
- **核心算法**：Dijkstra（优先实现，Floyd 可作为扩展对比）
- **前端原型**：HTML / CSS / JavaScript（纯静态，用于展示）
- **构建工具**：gcc 或 MinGW 编译
- **测试**：命令行手动测试 + 边界用例自动化脚本（如需要）

### 7.4 Assumptions

- 校园路径为双向道路，图使用无向图建模。
- 路径长度为整数（米）。
- 地点数量固定或支持预分配最大容量（如 MAX_NODES = 32）。
- 用户通过编号选择地点，而非输入中文名称。
- 前端原型仅用于演示和截图，C 程序为最终可运行程序。
