# course-design-report

**标签**: `ready-for-agent`  
**依赖**: test-cases, frontend-prototype

## 目标

按学校格式撰写课程设计说明书。

## 任务内容

1. 在 `docs/reports/` 下创建课程设计说明书文档（Word 或 Markdown）。
2. 说明书至少包含以下章节：
   - 封面（题目、学院、专业、姓名、学号、指导教师）
   - 需求分析
   - 概要设计（数据结构、算法选择）
   - 详细设计（模块说明、流程图）
   - 测试用例与结果
   - 总结与心得
   - 参考文献
3. 插入前端原型截图和 C 程序运行截图。
4. 参考 `docs/原始资料/课程设计任务书.txt` 中的格式要求。

## 验收标准

- [ ] 说明书结构完整，章节齐全。
- [ ] 包含需求分析、概要设计、详细设计、测试、总结。
- [ ] 包含至少 5 张系统截图。
- [ ] 参考文献格式正确。

## Review 要求

完成后提交文档，人工 review：
- 内容是否完整、符合学校格式
- 截图和表格是否清晰
- 文字是否有明显错误
