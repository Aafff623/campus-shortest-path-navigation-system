# final-review

**标签**: `ready-for-human`  
**依赖**: course-design-report

## 目标

在所有开发和文档工作完成后，进行最终人工 review，确保交付物齐全、质量达标。

## Review 清单

### 代码

- [ ] `src/` 下 C 代码结构清晰，能编译运行。
- [ ] Dijkstra 算法实现正确，测试通过。
- [ ] 异常输入处理完善，程序不崩溃。

### 文档

- [ ] `docs/reports/` 下课程设计说明书完整、格式正确。
- [ ] `docs/handoff/运行说明.md` 包含编译运行步骤。
- [ ] 截图齐全，能支撑说明书内容。

### 资料

- [ ] 任务书原件已归档到 `docs/原始资料/`。
- [ ] 前端原型已放入 `assets/prototype/`。
- [ ] PRD 和 Todo List 已更新状态。

### 提交

- [ ] 所有变更已提交到 Git。
- [ ] 提交历史清晰，符合 Conventional Commits。

## 完成后动作

1. 将本任务状态更新为 `completed`。
2. 打包所有交付物（源码 + 说明书 + 任务书）。
3. 按学校要求提交课程设计资料。

- 任务书 §6 设计成果形式及要求
