# test-cases

**标签**: `ready-for-agent`  
**依赖**: dijkstra-algorithm, input-validation

## 目标

编写并执行完整测试用例，验证系统功能与异常处理。

## 任务内容

1. 整理测试用例文档 `tests/test_cases.md`，包含：
   - 正常查询用例（至少 5 组不同起终点）
   - 异常输入用例（非数字、越界、相同起终点、不可达）
2. 在 `tests/` 下创建可运行的测试代码（可选）或手工测试记录。
3. 对比实际输出与预期输出，记录结果。
4. 如发现 bug，创建新的 `.scratch/` 任务修复。

## 验收标准

- [ ] `tests/test_cases.md` 包含 ≥5 组正常用例和 ≥4 组异常用例。
- [ ] 所有正常用例输出路径和距离正确。
- [ ] 所有异常用例得到预期提示，程序不崩溃。
- [ ] 测试结果截图或文字记录保存到 `docs/reports/`。

## Review 要求

完成后提交代码，人工 review：
- 测试用例是否覆盖主要路径
- 预期结果是否正确
- 是否有遗漏的边界情况
